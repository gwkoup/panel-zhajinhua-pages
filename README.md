# panel-zhajinhua-pages

本仓库用于归档和发布多个独立的 HTML 页面。这些页面本身不隶属于任何特定网站或域名，仅作为静态资源集合进行存储与版本管理。

## 项目简介

这是一个通用型 HTML 页面仓库，主要用途包括：

- 保存与整理各类独立功能页面
- 提供静态页面的版本回溯与归档
- 方便通过 GitHub Pages 或其他静态托管服务进行部署

仓库内页面彼此独立，不依赖后端服务，适合作为轻量级前端资源集合。

## 目录说明

```
panel-zhajinhua-pages/
├── index.html          # 主入口页面（如有）
├── pages/              # 存放各独立 HTML 页面的子目录
│   ├── example-1.html
│   ├── example-2.html
│   └── ...
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：主要页面归档目录，每个文件对应一个独立页面。
- `assets/`：存放页面共享的静态资源，按类型分类。
- 根目录下的 `index.html` 可作为导航页或默认起始页。

## 页面归档说明

- 所有页面均以 `.html` 格式保存，可使用浏览器直接打开预览。
- 页面内部资源引用使用相对路径，确保在本地或托管环境中均可正常加载。
- 仓库不包含任何动态脚本或数据库依赖，所有功能均在前端实现。
- 页面内容与样式遵循基本 Web 标准，兼容主流现代浏览器。

## 维护说明

- 新增页面：在 `pages/` 目录下创建新 HTML 文件，并更新 `index.html` 导航（如有）。
- 修改页面：直接编辑对应 HTML 文件，提交时请附带简要说明。
- 删除页面：移除文件并清理相关资源引用。
- 资源更新：如需添加或修改公共资源，请放入 `assets/` 相应子目录。

欢迎通过 Pull Request 或 Issue 提出改进建议或报告问题。
